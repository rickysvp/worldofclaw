# UI/UX Pro Max Skill

Advanced UI/UX design skill for creating professional, polished interfaces with comprehensive design systems.

## When to Invoke

- Building comprehensive design systems
- Creating component libraries
- Need professional-grade UI/UX
- Enterprise or SaaS application design
- Complex dashboard or admin interfaces
- Multi-platform design consistency

## Core Principles

### 1. Design System Architecture
- **Tokens**: Colors, typography, spacing, shadows as variables
- **Components**: Reusable, composable, accessible
- **Patterns**: Common UX patterns and flows
- **Documentation**: Clear usage guidelines

### 2. Visual Hierarchy
- **Scale**: Clear size differences between elements
- **Weight**: Font weights guide attention
- **Color**: Purposeful use of brand colors
- **Space**: Generous whitespace, clear grouping

### 3. Interaction Design
- **Feedback**: Every action has a reaction
- **Transitions**: Smooth, purposeful animations
- **States**: Hover, active, disabled, loading
- **Micro-interactions**: Delightful details

### 4. Accessibility
- **Contrast**: WCAG AA minimum
- **Focus**: Visible focus indicators
- **Semantics**: Proper HTML structure
- **Testing**: Keyboard navigation, screen readers

## Design Process

### Phase 1: Discovery
1. Understand user needs and goals
2. Analyze competitors and references
3. Define success metrics
4. Create user flows

### Phase 2: Structure
1. Information architecture
2. Wireframes (low fidelity)
3. Content strategy
4. Navigation design

### Phase 3: Visual
1. Mood boards
2. Color palette
3. Typography system
4. Component design

### Phase 4: Refinement
1. Prototype interactions
2. Usability testing
3. Iterate based on feedback
4. Final polish

## Component Guidelines

### Buttons
- Clear visual hierarchy (primary, secondary, tertiary)
- Consistent padding and sizing
- Loading and disabled states
- Icon + text combinations

### Forms
- Clear labels and helper text
- Validation and error states
- Progress indicators for multi-step
- Auto-save where appropriate

### Navigation
- Clear wayfinding
- Breadcrumbs for deep hierarchies
- Mobile-responsive patterns
- Active state indicators

### Data Display
- Tables with sorting/filtering
- Cards for content browsing
- Charts and visualizations
- Empty states

## Layout Patterns

### Dashboard
- Key metrics at top
- Charts and trends
- Recent activity
- Quick actions

### List/Grid
- Filtering and sorting
- Pagination or infinite scroll
- Bulk actions
- Search functionality

### Detail View
- Breadcrumb navigation
- Key information prominent
- Related content
- Action buttons

### Forms
- Section grouping
- Progress indication
- Validation feedback
- Save/Cancel actions

## Quality Checklist

### Visual
- [ ] Consistent spacing (8px grid)
- [ ] Typography hierarchy clear
- [ ] Color palette cohesive
- [ ] Icons consistent style
- [ ] Shadows and depth purposeful

### Interaction
- [ ] All states designed
- [ ] Loading states considered
- [ ] Error states handled
- [ ] Transitions smooth
- [ ] Feedback immediate

### Accessibility
- [ ] Color contrast checked
- [ ] Focus states visible
- [ ] ARIA labels where needed
- [ ] Keyboard navigable
- [ ] Screen reader tested

### Responsive
- [ ] Mobile layout designed
- [ ] Tablet breakpoints
- [ ] Desktop optimized
- [ ] Touch targets adequate
- [ ] Content reflows properly

## Output Format

1. **Design System Overview**
   - Color palette with variables
   - Typography scale
   - Spacing system
   - Component inventory

2. **Key Screens**
   - High-fidelity designs
   - Responsive variations
   - Interaction states

3. **Component Library**
   - Reusable components
   - Props and variants
   - Usage examples

4. **Documentation**
   - Design rationale
   - Usage guidelines
   - Do's and don'ts

## Best Practices

- Mobile-first approach
- Progressive disclosure
- Consistent patterns
- Performance conscious
