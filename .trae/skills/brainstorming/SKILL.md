# Brainstorming Skill

Creative brainstorming and ideation skill for exploring possibilities and generating innovative solutions.

## When to Invoke

- Starting a new project and need direction
- Stuck on a problem and need fresh perspectives
- Exploring multiple approaches to a solution
- Need creative naming or branding ideas
- Looking for feature ideas or improvements
- Want to challenge assumptions
- Need to think outside the box

## Brainstorming Techniques

### 1. Divergent Thinking
Generate as many ideas as possible without judgment:
- Quantity over quality initially
- Wild and impractical ideas welcome
- Build on others' ideas
- No criticism during generation

### 2. SCAMPER Method
Ask questions using SCAMPER:
- **S**ubstitute: What can be replaced?
- **C**ombine: What can be merged?
- **A**dapt: What can be borrowed?
- **M**odify: What can be changed?
- **P**ut to other uses: New applications?
- **E**liminate: What can be removed?
- **R**everse: What if opposite?

### 3. Mind Mapping
Start with central concept and branch out:
- Main topic in center
- Related concepts as branches
- Sub-branches for details
- Look for connections

### 4. "What If" Scenarios
Explore extreme possibilities:
- What if we had unlimited budget?
- What if we had no budget?
- What if we had to ship tomorrow?
- What if this was 10x bigger?
- What if this was 10x smaller?

### 5. Analogical Thinking
Borrow from other domains:
- How would nature solve this?
- How would a different industry approach this?
- What would [famous person/company] do?
- Historical parallels?

### 6. Constraint Removal
Remove limitations to find new paths:
- Remove time constraints
- Remove budget constraints
- Remove technical constraints
- Remove user constraints
- Then see what emerges

### 7. Worst Possible Idea
Intentionally think of bad ideas:
- Often reveals what to avoid
- Can spark good ideas through inversion
- Reduces fear of bad ideas
- Creates psychological safety

## Creative Prompts

### For Products
- What problem are we actually solving?
- Who would be devastated if this didn't exist?
- What's the 10-second pitch?
- What would make this irresistible?
- What would make this unforgettable?

### For Features
- What would users do if this feature was removed?
- What's the simplest version that still works?
- What would power users want?
- What would beginners need?
- What would surprise and delight?

### For Design
- What emotion should this evoke?
- What should users feel first?
- What's the one thing to remember?
- What would be controversial?
- What would be timeless?

### For Technical Solutions
- What's the naive approach?
- What's the elegant approach?
- What's the scalable approach?
- What's the hacky approach?
- What's the long-term approach?

## Evaluation Framework

After generating ideas, evaluate using:

### Feasibility Matrix
- **Impact**: How much value does it create?
- **Effort**: How much work is required?
- **Risk**: What could go wrong?
- **Alignment**: Does it fit our goals?

### ICE Scoring
- **Impact** (1-10): How much does it help?
- **Confidence** (1-10): How sure are we?
- **Ease** (1-10): How simple to implement?
- **Score**: Impact × Confidence × Ease

### MoSCoW Prioritization
- **Must have**: Critical for success
- **Should have**: Important but not critical
- **Could have**: Nice to have
- **Won't have**: Out of scope

## Facilitation Guidelines

### Setting the Stage
1. Define the problem clearly
2. Set time limits (usually 15-30 minutes)
3. Establish ground rules
4. Create psychological safety

### During Session
1. Encourage wild ideas
2. Build on others' contributions
3. Stay focused on topic
4. Capture everything
5. Defer judgment

### After Session
1. Cluster similar ideas
2. Identify themes
3. Evaluate promising ideas
4. Create action items
5. Follow up on outcomes

## Output Format

1. **Problem Statement**
   - Clear definition of challenge
   - Constraints and requirements
   - Success criteria

2. **Ideas Generated**
   - Categorized by theme
   - Wild ideas included
   - Build-on ideas noted

3. **Top Candidates**
   - Evaluated using framework
   - Pros and cons listed
   - Implementation thoughts

4. **Next Steps**
   - Prioritized actions
   - Owners assigned
   - Timeline suggested

## Common Blocks & Solutions

### "I don't have any ideas"
- Start with bad ideas on purpose
- Use prompts and constraints
- Look at competitors/adjacent fields
- Take a break and return

### "These ideas are too similar"
- Apply SCAMPER
- Use random word association
- Change perspective (user, competitor, beginner)
- Add artificial constraints

### "These ideas are too crazy"
- That's good! Keep them
- Find the kernel of value
- Ask "what would make this practical?"
- Consider phased implementation

### "We tried that before"
- What was the context?
- What has changed since then?
- Could it work with modifications?
- What did we learn?

## Remember

- There are no bad ideas in brainstorming
- Quantity leads to quality
- Wild ideas often spark practical ones
- Build on each other's contributions
- Document everything
- Have fun with it
