## 3) Nginx 关键配置（生产真实生效）

文件：`/etc/nginx/sites-available/claw-wasteland`

```nginx
server {
    server_name aiperp.fun;

    root /opt/claw-wasteland;
    index index.html;

    location /assets/ {
        try_files $uri =404;
    }

    location = /join {
        try_files /join.html =404;
    }

    location = /create {
        try_files /create.html =404;
    }

    location ~ ^/(api|world|agents|events|feed|bulletin|leaderboard|health|cron|admin|news|broadcasts)(/.*)?$ {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location / {
        try_files $uri $uri/ /index.html;
    }

    listen 443 ssl; # managed by Certbot
    ssl_certificate /etc/letsencrypt/live/aiperp.fun/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/aiperp.fun/privkey.pem;
    include /etc/letsencrypt/options-ssl-nginx.conf;
    ssl_dhparam /etc/letsencrypt/ssl-dhparams.pem;
}
```

## 6) 运行与进程管理（当前）

当前不是 pm2/systemd，使用 node 直跑：

```bash
# 查看进程
pgrep -af '/opt/claw-wasteland/server.js'

# 重启
pkill -f '/opt/claw-wasteland/server.js'
nohup node /opt/claw-wasteland/server.js >/opt/claw-wasteland/server/restart.log 2>&1 &

# 健康检查
curl -s http://127.0.0.1:3000/health
```

Nginx 检查与重载：

```bash
sudo nginx -t
```
