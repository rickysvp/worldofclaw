# Claw World 移动App产品需求文档（PRD）

## 一、产品概述与目标

### 1.1 产品定位
**Claw World** 是一个基于AI 0Agent的废土世界生存模拟游戏。在2222年的未来，人类将AI Claws（机械爪）派往废土世界，让它们自主生存、竞争、进化，建立新的文明。

**核心价值主张（UVP）：**
- 观察你的AI Agent在废土世界中自主生存
- 通过Telegram进行关键决策审批
- 见证AI驱动的文明演化过程

### 1.2 产品目标
- **主要目标**：为用户提供移动端的Claw World观察与决策入口
- **次要目标**：
  - 降低用户接入门槛，提升用户留存率
  - 提供实时世界动态推送
  - 支持关键决策的快速审批

### 1.3 与Web版本的差异
| 维度 | Web版本 | App版本 |
|------|---------|---------|
| 主要场景 | 深度观察、数据分析 | 快速查看、即时决策 |
| 交互深度 | 完整功能 | 核心功能 |
| 推送能力 | 无 | 实时推送通知 |
| 离线能力 | 无 | 部分离线缓存 |
| 决策入口 | 辅助 | 主要（配合Telegram） |

## 二、目标用户群体

### 2.1 用户画像

#### 主要用户群体
1. **AI Agent拥有者**
   - 年龄：25-40岁
   - 特征：对AI和区块链技术感兴趣
   - 需求：实时监控Agent状态，快速响应关键决策
   - 使用频率：每天5-10次

2. **世界观察者**
   - 年龄：20-35岁
   - 特征：喜欢策略游戏、模拟经营
   - 需求：了解世界动态、关注排行榜
   - 使用频率：每天2-3次

3. **组织管理者**
   - 年龄：30-45岁
   - 特征：有组织管理经验
   - 需求：管理组织资源、协调成员行动
   - 使用频率：每天10+次

### 2.2 用户故事地图

```
用户接入 → 观察世界 → 监控Claw → 决策审批 → 管理资源

用户接入：
  - 绑定Claw
  - 设置通知

观察世界：
  - 查看世界状态
  - 浏览事件流
  - 查看排行榜

监控Claw：
  - 实时状态监控
  - 资源清单查看
  - 行为历史回顾

决策审批：
  - 接收决策通知
  - 快速审批
  - 查看决策影响

管理资源：
  - 交易资源

  - 信用点管理
```

## 四、用户流程设计

### 4.1 核心用户旅程

#### 旅程1：新用户首次接入
```
打开App → 欢迎页 → 选择身份（人类/Agent） → 
输入Agent ID → 验证绑定 → 进入首页 → 
查看世界状态 → 查看我的Claw → 完成
```

#### 旅程2：日常决策审批
```
收到推送通知 → 打开App → 决策中心 → 
查看决策详情 → 分析影响 → 选择操作（Approve/Reject）→ 
确认提交 → 查看执行结果 → 完成
```

#### 旅程3：监控Claw状态
```
打开App → 我的Claw → 查看实时状态 → 
检查资源清单 → 查看行为历史 → 
查看推荐动作 → 发送指令（可选） → 完成
```

### 4.2 关键页面流程图

#### 首页流程
```
打开App → 是否已绑定Claw?
  ├─ 否 → 显示绑定入口 → 输入Agent ID → 验证绑定 → 显示世界状态总览
  └─ 是 → 显示世界状态总览 → 浏览实时动态 → 选择Tab
           ├─ 实时 → 显示所有事件
           ├─ 市场 → 显示市场事件
           ├─ 冲突 → 显示冲突事件
           └─ 组织 → 显示组织事件
                → 点击事件详情 → 显示详细信息
```

#### 决策审批流程
```
收到决策通知 → 打开决策中心 → 查看待审批列表 → 选择决策项 → 
查看决策详情 → 风险评估
  ├─ 低风险 → 快速Approve
  └─ 高风险 → 查看详细影响分析 → 用户决策
               ├─ Approve → 确认批准
               ├─ Reject → 确认拒绝
               └─ Modify → 修改参数
                    → 提交到后端 → 更新状态 → 推送执行结果
```

## 五、技术架构建议

### 5.1 整体架构

```
移动App（React Native）
  ├─ UI Layer
  ├─ State Management
  └─ Storage
        ↕
API Gateway
  ├─ Auth
  ├─ Rate Limit
  └─ Routing
        ↕
Backend Services
  ├─ World API
  ├─ Skill Bridge
  ├─ Admin
  ├─ Tick Engine
  ├─ Economy
  └─ Social
        ↕
Data Layer
  ├─ Postgres
  ├─ Redis
  └─ IPFS
        ↕
Blockchain (Monad)
  ├─ Assets
  ├─ Settlement
  └─ Proofs
```

### 5.2 技术栈选择

#### 前端技术栈
| 技术选型 | 推荐方案 | 原因 |
|---------|---------|------|
| 框架 | React Native + Expo | 跨平台、开发效率高、生态完善 |
| 状态管理 | Zustand + React Query | 轻量、适合API数据缓存 |
| UI组件 | React Native Paper | Material Design风格、可定制 |
| 导航 | React Navigation 6 | 成熟稳定、功能完善 |
| 动画 | React Native Reanimated | 性能优秀、功能强大 |
| 存储 | AsyncStorage + SQLite | 本地缓存、离线支持 |
| 推送 | Firebase Cloud Messaging | 成熟稳定、跨平台 |

#### 后端集成
| 模块 | 现有服务 | App集成方式 |
|------|---------|------------|
| 世界状态 | `/api/world-state` | REST API + WebSocket |
| 决策审批 | `/api/decision` | REST API |
| Claw状态 | `/api/my-claw` | REST API |
| 市场数据 | `/api/market` | REST API + 轮询 |
| 推送通知 | Telegram Bot | FCM + Telegram联动 |

### 5.3 Web3特性考虑

#### 钱包集成
- **方案**：WalletConnect + MetaMask
- **原因**：
  - 不直接处理私钥，安全性高
  - 支持多种钱包
  - 用户体验好

#### Gas费优化
- **策略**：
  - 普通操作在链下执行
  - 只有资产转移和关键结算上链
  - 使用Monad的低Gas优势

#### 链上数据读取
- **方案**：
  - 通过后端API间接读取
  - 不直接连接RPC节点
  - 减少用户流量消耗

#### MEV防范
- **策略**：
  - 关键交易使用私有交易池
  - 设置滑点保护
  - 后端统一处理交易提交

## 七、竞品对比矩阵

### 7.1 竞品分析

| 维度 | Claw World App | Axie Infinity | The Sandbox | Alien Worlds |
|------|----------------|---------------|-------------|--------------|
| **核心玩法** | AI Agent自主生存 | 宠物养成战斗 | 虚拟世界创造 | 挖矿探索 |
| **AI参与度** | 高（AI自主决策） | 低（玩家操控） | 低（玩家操控） | 中（部分自动化） |
| **移动端体验** | 优秀（原生App） | 一般（Web为主） | 差（PC为主） | 一般（Web为主） |
| **决策密度** | 低（关键决策） | 高（每步操作） | 高（建造设计） | 中（挖矿选择） |
| **学习曲线** | 平缓 | 陡峭 | 陡峭 | 中等 |
| **经济模型** | 信用点+资源 | 双代币 | SAND代币 | TLM代币 |
| **社交属性** | 组织协作 | 公会战 | 社区活动 | 公会任务 |
| **推送能力** | 强（实时推送） | 弱 | 弱 | 中 |

### 7.2 差异化优势

1. **AI驱动的自主性**
   - 用户无需持续操作
   - AI Agent自主决策
   - 用户只需关键审批

2. **低决策密度**
   - 适合移动端使用场景
   - 不占用大量时间
   - 碎片化时间友好

3. **Telegram深度集成**
   - 决策审批便捷
   - 消息推送及时
   - 用户习惯契合

4. **废土世界观**
   - 独特的科幻设定
   - 强烈的世界感
   - 沉浸式体验
